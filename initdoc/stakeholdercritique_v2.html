The five voices

What got fixed, what still itches

Each stakeholder is named in the challenge brief. Their daily job hasn't changed — so the test is whether the build now answers the question they actually bring to work.

1 · Provincial Policymaker

"Ik schrijf adviezen en brief Provinciale Staten. Ik ben geen hydroloog."

What now works

  • ✅ I can ask the tool what it knows and get sourced, plain-Dutch answers — my #1 ask in v1.
  • ✅ A defensible citation: APA string + Scenario-ID, hash, date, software commit, dataset versions, stable URL — plus a one-click PDF adviesnota.
  • ✅ Plain-language GO / CAUTION / STOP verdict before any number; official policy links attached with a disclaimer.

What still itches

  • 🟡 No always-visible "Kennisbasis" panel with dataset freshness dates — I can ask, but not glance.
  • 🟡 Stable URLs exist, but no browseable saved-scenario library to confirm a colleague and I opened the same calculation.

2 · Spatial Planner (Omgevingsdienst)

"200+ vergunningstoetsen open. Ik heb snelle, verdedigbare antwoorden nodig."

What now works

  • ✅ One-second GO/CAUTION/STOP on every plan, with the share of niet-haalbaar cells.
  • ✅ "Wat maakt dit haalbaar?" — ranked interventions, re-scored on real data, each with a source.
  • ✅ What-if sliders and a validated recipe-builder (never free-form code).

What still itches

  • ⛔ Cumulative effect: I still can't drop several concurrent plans and see their combined load on a zone.
  • ⛔ No "al vergund / in behandeling" demand layer — residual capacity doesn't net out approved permits (the operational dataset ships empty).

3 · Water Authority (Dunea / Evides / Oasen)

"Ik plan productiecapaciteit 10–25 jaar vooruit."

What now works

  • ✅ Per-intake chloride threshold mechanism (sourced 150 mg/L fallback, explicitly flagged — never a silent global constant).
  • ✅ Determinism I can cite in a dispute: parameters hash to a stable ID, results cache, dataset versions recorded, LLM at temperature 0.
  • ✅ Honest about the salinity signal being a saturated mask, amplified by the KNMI dryness knob.

What still itches

  • 🟡 Live Waterinfo (RWS) chloride still isn't mandatory for IJssel/Lek/Maas — a labelled fallback, not the live feed.
  • 🟡 Per-intake thresholds are mechanism-only until productieketen data lands.
  • ⛔ No directed production-chain flow view (intake → productie → distributie).

4 · Project Developer

"Ik wil weten of mijn project water krijgt. Geen wateropleiding."

What now works

  • ✅ Numbers I can feel: every headline metric carries a plain-Dutch, VEWIN-sourced human-scale analogy.
  • ✅ The feasibility traffic light + the "make it feasible" options before I buy land.
  • ✅ A stable, shareable scenario URL for a financing memo.

What still itches

  • ⛔ I can't see if another developer already claimed the headroom — two plans both read "green" because nothing nets them together.

5 · General Public / Citizen

"Ik woon in Zoetermeer en wil weten of mijn drinkwater veilig is in 2040."

What now works

  • ✅ A real citizen answer: verdict-first, jargon-free, <100 words, names my water company with phone + link, points to official policy, with a "verkenning, geen meting" disclaimer.
  • ✅ It won't quietly contradict the official line — every answer is anchored to the published Waterprogramma / KNMI position.

What still itches

  • 🟡 The citizen response exists in the engine, but a dedicated "burger" front-door view would close the loop.

Jury alignment

Against the Challenge Brief criteria

Moet (Must) · ✅ all met

Works on the supplied data; answers exploratory what-if questions combining ≥2 themes (verzilting × CBS × zes-uurszones); open-source with READMEs; the original descriptive flow still works untouched.

Zou moeten (Should) · ✅ / 🟡

Two+ scenarios + a with/without-shock comparison; uncertainty, assumptions and time-horizon shown explicitly; ambiguous input gets a clarifying question. Navolgbaar: Insight trail + chatbot citations done; MLflow scenario tracing still deferred.

Zou niet (Should not) · ✅ respected

The LangGraph / DuckDB / frontend architecture was extended, not rebuilt; the reasoning chain is never hidden behind a single answer.

Kan (Could) · ✅ / 🟡

A dedicated scenario node composing shocks; a sliders/presets UI (the recipe-builder). Explicit FDS / NORA alignment not yet documented.

The project's own v3 re-grade scored 34/46 implemented before the chatbot existed; the chatbot advances Gap 1 and the headline "query-anything-with-sources" capability beyond that grade.

What's left

Consolidated residual gaps, prioritised

  1. Cumulative / multi-project overlay + "al vergund" layer — the deepest remaining silo gap (planner, developer, water authority). Needs a committed-demand data source.
  2. Bring source-citations to the descriptive SQL assistant — so every query surface, not just the new chatbot, answers with hyperlinks. Highest-leverage transparency fix.
  3. Live Waterinfo (RWS) chloride for IJssel/Lek/Maas intake scenarios — mandatory, with an explicit "laatste bekende waarde van [datum]" warning on failure.
  4. Always-visible Kennisbasis + freshness panel and a permanent plain-language map title — finish the glanceable half of the intuition pointer.
  5. MLflow scenario tracing — closes the last "navolgbaar" axis for the jury.
  6. Shared scenario-library UI + a /verify re-run — turn the existing stable URLs into a browseable, re-verifiable institutional library.

Verdict

The remediation landed. Six of the ten gaps are fully fixed, three partial, one open — and the headline "ask anything, get sourced answers" requirement is now real through the knowledge chatbot. Silos are broken by shared, reproducible, sourced outputs; consistency is hard-guaranteed for scenarios; transparency is strong end-to-end. What remains is additive, not corrective: the cumulative-effect overlay (data-dependent), citations on the legacy SQL path, live Waterinfo, MLflow tracing, and a few glanceable UI panels. For a non-technical policy audience the tool is now legible, defensible and citeable.
