Part B · the policymaker speaks

The gaps nobody has listed yet

"The tool is now genuinely good — legible, sourced, reproducible. So let me be the awkward voice and ask the questions that surface the first time this carries a real advies."

A · Nobody can prove who ran this BLOCKER · Woo

There is no real authentication — the user is a stub, and nothing records who ran a scenario or changed an assumption. The moment an output lands in an official advies, the Wet open overheid and basic accountability demand a human-attributable trail. The tool has perfect calculation provenance and zero human provenance. This is a bigger production blocker than any data gap.

Fix direction: provincial SSO, a per-user action log (ran / shared / promoted / overrode), and the user identity stamped into the citation.

B · A single number where there should be a confidence band FALSE PRECISION

The verdict is a point estimate: "STOP at 66.1" and "CAUTION at 65.9" look identical to the model but flip the advice. Yet the assumptions carry ranges (chloride 150–250, demand 0.10–0.14, growth 0.04–0.12). Uncertainty is shown for inputs but not for the output, so a policymaker defending a STOP can't say how robust it is.

Fix direction: sweep the assumption ranges + the four KNMI scenarios and report robustness — "STOP in 3 van 4 klimaatscenario's" — committing only when the band is one-sided.

C · Who keeps the data fresh? GOVERNANCE

The build now detects drift and shows freshness dates — but there is no process: no refresh SLA, no owner, no escalating staleness banner. A 2022 dataset silently underpinning a 2026 advies is a credibility risk even when the date is technically visible.

Fix direction: a per-dataset refresh owner + SLA, a staleness banner that escalates, and a scheduled refresh job.

D · Privacy asymmetry: the chat remembers everything AVG / GDPR

Phase 3 carefully anonymises PII before caching FAQs — but the descriptive chat sessions store raw user questions (a citizen's postcode or address) in Postgres, with no anonymisation or retention policy. The careful door is locked while the window is open.

Fix direction: anonymise-on-write for session storage, a retention window, and a "geen persoonsgegevens invoeren" notice on the citizen path.

E · The model has never been checked against reality CALIBRATION

Every number is an assumption-driven composite — sourced and reproducible, but never validated against an actual Dunea/Evides capacity forecast or a published drinkwater study. The disclaimer is honest, yet a jury or director will ask: "has this verdict ever matched a real assessment?" Without a calibration story the tool risks being confidently wrong.

Fix direction: a calibration appendix — run the engine against one or two known cases, report agreement, and state the validation status on the card.

F · Assumptions change; old adviezen shouldn't silently change with them REPRODUCIBILITY

Assumptions are sourced today, but when weights/thresholds are revised next quarter, re-opening a 2-year-old scenario URL may compute a different verdict — and the old advies cited the old model. The scenario-hash pins parameters, not the assumption library version.

Fix direction: version the assumption library (a methodology changelog + an assumptions_version in the citation), so "reproduce exactly as advised on [date]" is possible.

Honourable mentions: WCAG/accessibility isn't asserted for the citizen path (verdicts do use colour and text — good); and LLM cost / rate-limit / abuse controls aren't visible for a public-facing tool.

The one line for the steering group

The tool has crossed from "interesting prototype" to "defensible instrument." But three things still stand between it and a real advies: a human accountability trail (who), an uncertainty band (how sure), and a calibration story (is it right). Those — not more features — are the next milestone.
